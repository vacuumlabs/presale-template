# product-management/

Personas, PRDs, user journeys, feature maps, product strategy, domain briefs, competitive briefs, problem framing.

## Typical wiki pages

- `problem-statement.md` — the sharpest version of the problem we're solving
- `personas/<name>.md` — one page per persona. When the canonical source is a polished PO-authored file under `team-inputs/`, the wiki page references it via `canonical_source:` metadata rather than duplicating
- `pain-points.md` — synthesised pain points across personas and user research
- `journeys/<name>.md` — user journey maps
- `features/<name>.md` — one page per major feature or capability (what it does, why it matters, which persona it serves)
- `competitors/<name>.md` — one page per competitor (positioning, strengths, weaknesses, signals we've observed)
- `domain-brief.md` — the domain context (regulation, vocabulary, stakeholders outside the client)
