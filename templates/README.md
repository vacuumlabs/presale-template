# Templates

Structured inputs and outputs for the presale process. These are schema-layer files — never edit them per-engagement.

**Usage pattern:** Copy the relevant template to `team-inputs/`, fill it in, then run `/ingest team-inputs/<filename>.md` to integrate it into the wiki.

---

## Template reference

| Template | Stage | Owner | How to use |
| --- | --- | --- | --- |
| [T1 — Sales Deal Brief](T1-sales-deal-brief.md) | Intake (before G1) | Sales | Copy to `team-inputs/`, fill in before the intake call. Required input for `/p1`. |
| [T2 — Discovery Q&A](T2-discovery-qa.md) | Problem Framing (Stage 3) | PO | Copy to `team-inputs/`, fill in during/after discovery calls. Required input for `/p3`. |
| [T4 — Deal Outcome](T4-deal-outcome.md) | Debrief (Stage 9) | PM | Reference structure. `/p9` produces `_deal-outcome.md` at the repo root. |
| [T5 — Proposal Document](T5-proposal-document.md) | Proposal Assembly (Stage 7) | PM | Reference structure for the Google Doc. `/p7` drafts all sections; PM assembles into GDoc. |
| [T6 — Post-Mortem Call](T6-postmortem-call.md) | Debrief (Stage 9) | PM | Copy to `team-inputs/`, fill in during post-mortem call. Required input for `/p9`. |
| [T7 — Sales Strategy Alignment Brief](T7-sales-strategy-alignment.md) | Problem Framing → G2 (T2/T3 only) | PM | Copy to `team-inputs/`. Run `/p3f prepare` before the call; `/p3f document` after. Required input for `/p4`. |

---

## Notes

- **T5** is a reference structure only — it guides what AI-generated output looks like, not a file you fill in by hand.
- **T1, T2, T6, T7** are working documents you fill in and then ingest.
- **T4** is generated by `/p9` — you don't fill it in manually.
- **The ARB submission template lives on Confluence**, not in this repo. `/p6` fetches it at runtime from [page 3109978115](https://vacuum.atlassian.net/wiki/x/A4BeuQ) and writes the draft to `technical-architecture/arb-records/YYYY-MM-DD-arb-submission.md`.
- Templates are versioned and propagated via `/sync-template`. Do not edit per-engagement; copy to `team-inputs/` first.
